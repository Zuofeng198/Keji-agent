# 离线 / 免下载部署

不必在每台电脑上都从 PyPI 下载，可以**在本机先打好包**，再和项目一起拷走。

## 三种方式对比

| 方式 | 体积 | 目标机要求 | 说明 |
|------|------|------------|------|
| **A. 离线 wheel 包**（推荐） | 约 1～3 GB | 装 **与 BUILD_INFO 相同主版本** 的 Python（如 3.10.x） | 3.12 机不能用 3.10 的 wheel，需用 3.12 重打 |
| **B. 复制整个 venv** | 约 3～8 GB | 同版本 Windows 64 位 + **同版本 Python** | 最快，路径/版本要一致 |
| **C. 在线 pip** | 最小 | Python + 能上网 | `setup_deploy.bat` 默认 |

## A. 推荐：离线 wheel（本机执行一次）

在**已能正常运行**的开发机上：

1. 双击 **`package_offline.bat`**，选 **1**（仅 wheel，体积较小）  
   或选 **2**（wheel + 复制 venv，目标机几乎不用 pip）

2. 生成目录：
   - `offline_packages/pip_wheels/` — 所有依赖安装包
   - `offline_packages/BUILD_INFO.txt` — 记录 Python 版本，**目标机主版本必须一致**（当前准备版为 **3.12**）

   指定版本打 wheel：`powershell -File scripts\package_offline.ps1 -PythonVersion 3.12`  
   或双击 **`package_wheels.bat`**（已默认 3.12）。

3. 拷贝到新机：
   - 整个项目文件夹（含 `offline_packages`）
   - 若选了 2，还要带上 **`portable_bundle/`**

4. 新机双击 **`setup_deploy.bat`**  
   - 有 `pip_wheels` 时：**不从网上下载**，本地安装  
   - 有 `portable_bundle/venv` 时：直接还原 venv

新机仍需安装 **Python**（安装包很小），一般**不需要联网 pip**。

## B. 直接复制 venv 的注意点

- 开发机与目标机：**Windows 64 位、Python 主版本一致**（如都是 3.12.x）  
- 建议两边项目放在**相同盘符路径**（例如都是 `D:\科吉agent`）  
- `package_offline.bat` 选 **2** 会生成 `portable_bundle/venv`

## Node / MCP 离线（可选）

`package_offline.bat` 选 **3** 会额外复制 `node_modules` 到  
`offline_packages/node_modules_root/`，目标机 **`setup_deploy.bat`** 会自动还原，无需 `npm install`。

目标机仍要装 **Node**（若要用 npx 启动的 MCP），但可以不联网下载 node 包。

## 准备版目录建议

```
科吉agent准备版/
  项目源码...
  offline_packages/    ← 本机 package_offline 生成后拷入
  portable_bundle/     ← 可选，选 2 或 3 时才有
```

`.gitignore` 已忽略 `offline_packages/`，避免误提交巨大文件。

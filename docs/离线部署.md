# 离线 / 内网部署

适合：**不能访问 PyPI** 或希望多台机器**统一依赖版本**。

本 GitHub 仓库已包含 **Python 3.12 / Windows 64 位** 的 wheel（`offline_packages/pip_wheels/`，通过 **Git LFS** 分发）。克隆后执行 `git lfs pull` 即可，无需在本机重新打包。

## 克隆后直接部署

```powershell
git clone https://github.com/Zuofeng198/keji.git
cd keji
git lfs install
git lfs pull
setup_deploy.bat
```

`setup_deploy.bat` 会优先使用 `offline_packages/pip_wheels`，**不联网 pip**。

## 自行重新打包 wheel

仅在需要 **更换 Python 版本**（如改为 3.11）时：

1. 在本机用目标版本 Python 创建/使用 `venv`
2. 双击 **`package_wheels.bat`**
3. 将生成的 `offline_packages/` 随项目拷到目标机

查看版本记录：`offline_packages/BUILD_INFO.txt`

## 三种部署方式对比

| 方式 | 说明 |
|------|------|
| **Git + LFS（当前仓库）** | 克隆 + `git lfs pull` + `setup_deploy.bat` |
| **U 盘拷贝整个文件夹** | 含 `offline_packages` 即可 |
| **仅源码、无 offline** | `setup_deploy.bat` 自动联网 pip（需能上网） |

## Node / MCP

- 不装 Node：核心对话、团队文件仍可用，部分 MCP 不可用  
- 完全离线 Node 包：在开发机运行 `package_offline.bat` 选 **3**，会生成 `offline_packages/node_modules_root/`

## 注意

- wheel **不能跨主版本**（3.12 的包不能给 3.10 用）
- `torch` 等大文件通过 Git LFS 存储，克隆务必安装 Git LFS

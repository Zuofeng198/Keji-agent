# 科吉agent 改造记录

## 概述
从 openAgent 开源项目移植 SQL 智能问数能力 + 全面前端优化

---

## 1. 新增功能

### 1.1 数据库连接与查询工具
- **文件**: `core/db_tools.py`
- **来源**: 从 openAgent 的 `mysql_mcp.py` / `postgresql_mcp.py` 移植改造
- **功能**: 6个注册工具，AI 对话中可直接调用
  - `db_connect` — 连接 MySQL/PostgreSQL
  - `db_list_tables` — 列出所有表
  - `db_describe_table` — 查看表结构
  - `db_execute_query` — 执行 SQL 查询
  - `db_test_connection` — 测试连接
  - `db_disconnect` — 断开连接

### 1.2 智能问数（NL2SQL）
- **文件**: `core/smart_query.py`
- **来源**: 从 openAgent 的 `smart_db_workflow.py` 移植
- **管线流程**:
  ```
  用户提问 → LLM 分析表结构 → 自动生成 SQL → 执行查询 → LLM 总结结果
  ```
- **API 端点**: `POST /api/smart-query` / `POST /api/smart-query/stream`

### 1.3 数据库配置管理
- **数据表**: `database_configs` + `table_metadata`（SQLite）
- **API 端点**: 11 个 RESTful 接口（CRUD + 测试连接 + 扫描表结构 + 元数据管理）
- **密码加密**: 使用 `cryptography.Fernet` 基于机器密钥加密存储

### 1.4 前端数据库页面
- 左右分栏布局：左主（3列网格数据源）+ 右侧栏（聊天风格智能问数）
- 思考面板（与主对话同款 `🧠 分析过程` 可折叠面板）
- 双主题兼容（Emoji / Font Awesome `.theme-fa`）

## 5. 踩坑记录

### 5.1 页面空白 — HTML 嵌套错误
**现象**: 点击数据库页面，主内容区空白，JS 无报错
**根因**: `page-database` div 被嵌套在 `page-settings` div 内部（settings 页面缺一个 `</div>` 闭合）
**定位方法**: F12 → Elements 看 DOM 层级 → 发现父元素是 `DIV.page` 而非 `MAIN`
**修复**: 补上 settings 页面缺的 `</div>`
**教训**: 在现有 HTML 中插入新页面，必须用 `section.count('<div') - section.count('</div>')` 检查 div 平衡

### 5.2 API 返回 400 — 旧进程残留
**现象**: 改了 Python 路由代码后，API 始终返回 400 Bad Request
**根因**: `desktop.py --mode web` 后台启动一直失败（exit 127），跑的还是旧服务进程
**定位方法**: `netstat -ano | findstr :8000` 发现进程 PID 没变，`TestClient` 测试成功但 curl 失败
**修复**: `taskkill /F /PID <pid>` 强杀后重新用 `uvicorn` 启动
**教训**: 改 Python 文件后必须确认新进程已启动（检查 PID），不能只看 health 响应

### 5.3 FastAPI dict 类型参数
**现象**: `data: dict` 参数在某些 FastAPI 版本中无法解析 JSON body
**修复**: 改用 Pydantic 模型 `class SmartQueryRequest(BaseModel): query: str; config_id: int`

### 5.4 中文乱码
**现象**: MySQL 显示 `å¼ ä¸‰` 等乱码
**根因**: 通过 bash `docker exec` 插入数据时编码不一致
**修复**: 用 Python pymysql 以 `utf8mb4` 重新插入

### 5.5 JS 函数跨 script 块
**现象**: 数据库 JS 函数定义在第三个 `<script>` 块，`switchPage` 调用时找不到
**修复**: 将所有数据库 JS 移到主 `<script>` 块（第一块）

### 5.6 Background 任务启动失败
**现象**: 所有 `run_in_background=true` 的 bash 任务 exit 127
**根因**: Bash 在后台模式下找不到 venv 的 Python 路径
**修复**: 改用前台 `uvicorn main:app` 启动

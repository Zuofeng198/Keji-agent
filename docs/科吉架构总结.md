## 数据流简图

```
用户 → index.html → sendChat() → POST /chat/stream
                                        │
                                  adapter.chat_stream()
                                        │
                                  AgentRunner.run(ReAct循环)
                                        │
                              ┌─────────┴──────────┐
                              ▼                    ▼
                         LLM (ollama)          Tool执行
                              │                    │
                              └─────────┬──────────┘
                                        ▼
                                   SSE事件流
                                   │
                          think_token → 思考面板 + 状态条
                          answer      → AI回答气泡
                          done        → 清理
```

## SSE 事件类型（/chat/stream）

| 事件 | 触发时机 | 前端处理 |
|:-----|:---------|:---------|
| `think_token` | LLM思考/工具调用（工具调用信息嵌在文本里） | 更新thinking-panel + status-bar |
| `answer` | 最终回答（每2个字符一个事件） | 追加到AI回答气泡 |
| `error` | 出错 | toast提示 |
| `done` | 流结束 | 清理所有badge + 停止计时器 |

> ⚠️ **注意**：工具调用**没有独立SSE事件**！tool_call/tool_result事件在 `/chat/stream` 路径上不存在。所有工具调用信息通过 `think_token` 事件以文本 "🔧 调用工具: xxx" 发送。前端通过解析 token 内容来识别工具执行状态。

# 数据库功能 — 前端空白问题排查记录

## 现象
新增「数据库」导航页后，点击侧边栏「数据库」，主内容区完全空白，无任何渲染内容。其他页面（对话、知识库、文件浏览、设置）均正常。

## 排查历程

### 1. 后端 API 验证 ✅
- 所有数据库相关 API 端点正常响应（`/api/database/configs`、`/api/smart-query` 等）
- 后端服务健康检查通过

### 2. HTML 页面结构验证 ✅
- `page-database` div 存在于 HTML 中
- JavaScript 函数全部定义在正确的 script 块中
- 无 JS 语法错误（Node.js 语法检查通过）

### 3. CSS 渲染验证 ✅
- `.page.active { display: flex; overflow-y: auto; }` — 正确覆盖 `.page { display: none; }`
- 添加 `overflow-y: auto` 防止内容溢出裁剪

### 4. JavaScript 函数执行验证 ✅
- 控制台输出确认 `switchPage('database')` 正确执行
- `target.style.display = 'flex'` 成功设置（改用手动设置 `style.display` 后生效）

### 5. DOM 层级追踪 🔴 找到根因
浏览器 F12 控制台输出 parent chain：
```
DIV#page-database.page rect=0x0
  -> DIV#page-settings.page rect=0x0
  -> MAIN.main rect=714x732
  -> BODY -> HTML
```

**`page-database` 是 `page-settings` 的子元素！** 不是 `main` 的直接子元素。

由于 `page-settings` 被 `switchPage` 设为 `display:none`，其内部的 `page-database` 也是 `0x0`，即使给 `page-database` 设置 `style.display='flex'` 也无济于事。

## 根因

**HTML 结构嵌套错误**：数据库页面 `<div class="page" id="page-database">` 被插入到了设置页面 `<div class="page" id="page-settings">` 的内部，而非 `main` 的直接子级。原因是设置页面结尾缺少一个 `</div>` 闭合标签。

修复方式（`web/index.html` 第 1808 行）：
```html
<!-- 修复前 -->
    </div>
  </div>

<!-- 修复后 — 补一个 </div> 闭合设置页 -->
    </div>
  </div></div>
```

## 附带改进

- `switchPage()` 函数改为同时设置 `style.display` 内联样式，不单纯依赖 CSS class，增强鲁棒性
- `.page.active` 增加 `overflow-y: auto` 避免长内容被裁剪
- 数据库相关 JS 函数移入主 `<script>` 块，消除跨块作用域问题

## 教训

1. 浏览器 F12 → Elements 检查 DOM 层级是定位 "页面空白" 最有效的方法
2. JS 没报错不代表 DOM 结构正确
3. **在已有 HTML 中插入新页面时，必须确认前后 `</div>` 的正确闭合**
4. 用 Python 统计 `<div>` 和 `</div>` 的差值能快速发现缺失：`section.count('<div') - section.count('</div>')`
5. 从备份文件恢复后，原 settings 页面就缺少一个 `</div>`，这是一个持续存在的 HTML 结构缺陷

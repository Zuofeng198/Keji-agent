# 科吉 Agent — 使用指南

面向**最终用户**与**部署管理员**，说明登录后怎么用、怎么管。

## 1. 打开系统

- 本机：http://127.0.0.1:8000/
- 局域网：http://**安装科吉的那台电脑IP**:8000/

若打不开，先确认服务器上已运行 `launch_keji.bat` 或 `run_server.bat`，且防火墙允许 8000 端口。

## 2. 登录

1. 打开网页后会看到登录界面（已关闭「免登录」时）。
2. **管理员**首次账号来自部署时的配置：
   - 用户名：默认 `admin`（见 `config.yaml` → `security.bootstrap_admin.username`）
   - 密码：`.env` 里的 `KEJI_ADMIN_PASSWORD`
3. 登录失败：检查密码是否改对、服务是否重启过。

## 3. 主界面

- **对话**：输入问题发送；左侧可切换/新建会话，历史按当前登录用户保存。
- **团队文件**：浏览/上传共享目录与个人目录（权限见下表）。
- **设置**：模型、API Key 等（视版本开放项而定）。
- **管理**（仅管理员）：用户列表、重置密码、查看他人会话摘要等。

## 4. 角色说明

| 角色 | 能做什么 | 不能做什么 |
|------|----------|------------|
| 管理员 | 全部功能、所有目录、用户管理 | — |
| 成员 | 对话写入工具；读写共享+个人文件夹 | 访问他人私人目录 |
| 只读 | 对话只读类工具；读共享+个人文件 | 创建/删除文件、执行写入类工具 |

管理员在「管理」页可新建用户并指定角色。

## 5. 团队文件

目录结构（在服务器磁盘上）：

```text
data/workspace/
  shared/          ← 所有人可见（只读角色不可写）
  users/
    <用户ID>/      ← 仅本人 + 管理员可访问
```

网页「团队文件」页可切换：**共享** / **我的**；管理员还可查看他人目录。

## 6. 对话与工具

- 成员、管理员可使用完整工具（写文档、跑代码、查知识库等，视配置启用）。
- 只读账号若要求「建文件夹、改 Excel」，会被系统拒绝并提示权限不足。
- 复杂任务首次可能稍慢（后台加载 MCP）。

## 7. 管理员常用操作

1. **添加成员**：管理 → 新建用户 → 选择角色 `member` 或 `readonly`。
2. **改密码**：管理 → 用户 → 重置密码；或运行 `scripts/reset_password.py`（需服务器上操作）。
3. **看会话**：管理 → 选择用户查看会话列表与预览（审计用途）。

## 8. 日常维护

| 事项 | 建议 |
|------|------|
| 备份 | 定期备份 `data/`（含 `keji.db` 与工作区文件） |
| 升级 | `git pull` + `git lfs pull` + 再运行 `setup_deploy.bat` |
| 日志 | 查看项目下 `logs/` |
| 密钥 | 只改 `.env`，勿提交到 Git |

## 9. 故障排查

| 现象 | 处理 |
|------|------|
| 网页无法访问 | 确认服务在跑、端口 8000、防火墙 |
| 登录立即掉线 | 检查 `.env` 中 `KEJI_JWT_SECRET`；清浏览器缓存重登 |
| 对话无回复 | 检查 `DEEPSEEK_API_KEY` 是否有效、余额 |
| 工具报权限 | 确认账号角色；只读不能用写入工具 |
| 团队文件 403 | 路径不在该用户允许的工作区内 |

部署相关问题见 [新机部署.md](新机部署.md)、[离线部署.md](离线部署.md)。

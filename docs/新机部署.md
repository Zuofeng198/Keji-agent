# 科吉 Agent — 新机一键部署

## 准备

1. 将整个项目文件夹拷到目标机（如 `D:\keji` 或 `科吉agent准备版`）
2. 安装 **Python 3.10～3.12**（64 位，勾选 Add to PATH）。若用离线 wheel，版本须与 `offline_packages/BUILD_INFO.txt` 一致
3. 建议安装 **Node.js 18+**（MCP 工具可选，不装也能先跑网页）

## 一键部署

双击 **`setup_deploy.bat`** 或 **`一键部署.bat`**。

若已在本机打好离线包（见 **`docs/离线部署.md`**），拷贝时带上 **`offline_packages/`**（可选 **`portable_bundle/`**），新机**可不用联网下载**依赖。

脚本会自动：

- 创建 `venv`（或从 `portable_bundle` 还原）
- 安装依赖：优先本地 `offline_packages/pip_wheels`，否则联网 pip（约 5～15 分钟）
- 无 `config.yaml` / `.env` 时从模板复制
- 自动生成 `KEJI_JWT_SECRET`
- 创建 `data/workspace`、`logs` 等目录

## 部署后必做

编辑项目根目录 **`.env`**，至少填写：

```env
DEEPSEEK_API_KEY=你的密钥
KEJI_ADMIN_PASSWORD=首次管理员密码
```

（若用 OpenAI，再填 `OPENAI_API_KEY`。）

## 启动

| 文件 | 说明 |
|------|------|
| `launch_keji.bat` / `启动科吉.bat` | 后台启动 + 自动打开浏览器 |
| `run_server.bat` / `运行服务.bat` | 黑窗运行，便于看日志 |

浏览器访问：**http://127.0.0.1:8000/**  
局域网其他电脑：**http://服务器IP:8000/**

## 关闭服务

任务管理器结束 **pythonw.exe** / **python.exe**，或关闭 `run_server.bat` 黑窗（Ctrl+C）。

## 常见问题

- **pip 安装失败**：确认 Python 64 位；可改用国内源：  
  `venv\Scripts\pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple`
- **环境变量警告**：`.env` 未填齐时出现，补全密钥后重启即可。
- **无 Node**：仅部分 MCP 不可用，核心对话与团队文件仍可用。

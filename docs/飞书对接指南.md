# 飞书渠道对接指南

## 前置条件

- 一个飞书账号（个人版即可，免费）
- 一台能运行科吉的电脑（需要能访问 `open.feishu.cn` 和 `wss://open.feishu.cn`）

---

## 第一步：飞书开放平台注册应用

1. 打开 [飞书开放平台](https://open.feishu.cn)（建议用电脑浏览器）
2. 点击右上角 **「开发者登录」**，用你的飞书手机 App 扫码登录
3. 登录后点击 **「创建应用」** → **「企业自建应用」**

   > ⚠️ 如果提示"暂无可管理租户"，点 **「创建测试企业」**，填一个企业名称即可。  
   > 测试企业不需要营业执照，完全免费，你就是这个企业的管理员。

4. 填写应用信息：
   - 应用名称：`科吉助手`（可随意取名）
   - 应用描述：`企业 AI 助手`

5. 创建成功后会自动跳转到应用管理页面。

---

## 第二步：获取凭证（App ID & App Secret）

1. 在左侧菜单找到 **「凭证与基础信息」**
2. 页面中可以看到：
   - `App ID`（格式如 `cli_xxxxxxxxxxxxxxxxxxxx`）
   - `App Secret`（一串密文）

3. **复制这两个值**，后续要填到 config.yaml 中。

---

## 第三步：启用机器人能力

1. 左侧菜单 → **「机器人」**
2. 打开 **「启用机器人」** 开关
3. 这一步不需要其他配置，默认即可

---

## 第四步：配置事件订阅

1. 左侧菜单 → **「事件订阅」**
2. **订阅方式选择「WebSocket 长连接」**（不需要填回调 URL，不需要域名，不需要 HTTPS）
3. 点击 **「添加事件」**
4. 搜索并勾选：`im.message.receive_v1`（接收消息事件）
5. 点击 **「确认添加」**

---

## 第五步：添加权限

1. 左侧菜单 → **「权限管理」**
2. 点击 **「添加权限」**，搜索并添加以下权限：

| 权限标识 | 用途 |
|---------|------|
| `im:message` | 读取用户发给机器人的消息 |
| `im:message:send_as_bot` | 以机器人身份发送消息 |
| `im:resource` | 下载图片和文件（可选，如果要用 OCR/文件处理） |
| `im:chat` | 获取群聊信息（可选，如果要在群聊中使用） |

---

## 第六步：发布应用

1. 左侧菜单 → **「版本管理与发布」**
2. 点击 **「创建版本」**
3. 填写：
   - 版本号：`0.0.1`
   - 更新说明：`初始版本`
4. 点击 **「保存」**
5. 回到版本列表，点击 **「申请发布」**

由于你是测试企业的管理员，审批人会是你自己。直接在审批页面点 **「同意」** 即可。

---

## 第七步：配置科吉

### 安装飞书 SDK

```bash
pip install lark-oapi
```

### 修改 config.yaml

打开 `config.yaml`，找到文件末尾的飞书配置段，填入第二步获取的 App ID 和 App Secret：

```yaml
channels:
  feishu:
    enabled: true                      # 改为 true 启用
    app_id: "cli_xxxxxxxxxxxxxxxxxxxx"  # 替换为你的 App ID
    app_secret: "xxxxxxxxxxxxxxxxxxxx"  # 替换为你的 App Secret
    encrypt_key: ""                     # 保留空（WebSocket 模式不需要）
    verification_token: ""              # 保留空
    allow_from: ["*"]                   # ["*"]=允许所有人（上线后可改成具体用户ID）
    streaming: true                     # 飞书打字机效果回复
    group_policy: mention               # 群聊仅 @科吉 触发
    domain: feishu                      # 国内飞书保持 feishu
```

### 启动科吉

```bash
python main.py
# 或双击 启动科吉.bat
```

启动日志中应该能看到：
```
飞书桥接层已启动，等待 WebSocket 连接...
Feishu bot started with WebSocket long connection
No public IP required - using WebSocket to receive events
```

---

## 第八步：在飞书手机端使用

1. 打开手机飞书 App
2. 点击顶部的搜索框 / 全局搜索
3. 搜索 **「科吉助手」**（第二步填的应用名称）
4. 点击搜索结果中的机器人
5. 进入聊天窗口，直接打字即可

### 测试

```
你：你好
科吉：你好！我是科吉，你的企业 AI 助手。有什么可以帮你的？
```

```
你：帮我分析一下桌面上的文件，给我一个分类统计
科吉：好的，正在扫描桌面文件...
     （打字机效果逐字出现结果）
```

---

## 群聊使用

1. 在企业群聊中点击 **「+」** → **「添加机器人」**
2. 搜索「科吉助手」并添加
3. 在群里 `@科吉助手 帮我...` 即可触发

`group_policy: mention` 配置确保：
- 群聊中只有 `@科吉助手` 时才响应
- 不会干扰群里的正常聊天

## 常见问题

### Q: 启动时报 "Feishu SDK not installed"

```bash
pip install lark-oapi
```

### Q: 启动时报 "飞书渠道配置不完整"

检查 config.yaml 中 `app_id` 和 `app_secret` 是否已填写，**不要留空字符串**。

### Q: 飞书提示 "应用尚未发布" 或 "找不到应用"

回到飞书开放平台 → 版本管理与发布 → 确认已经申请发布并通过审批。

### Q: 消息发出去没有回复

1. 检查科吉是否正常启动
2. 检查日志中是否有 WebSocket 连接信息
3. 检查 allow_from 是否设置为 `["*"]`（测试阶段）
4. 检查网络是否能访问 `wss://open.feishu.cn`（公司内网可能需要放开防火墙）

### Q: 是否需要公网 IP？

**不需要。** 飞书渠道使用 WebSocket 长连接，由飞书服务器主动推送消息，科吉只需要能**出站**访问 `wss://open.feishu.cn` 即可，不需要公网 IP，不需要内网穿透，不需要 HTTPS 证书。

---

## 非覆盖式原则

飞书渠道的接入**完全不影响**现有的 Web 聊天功能。浏览器访问 `http://127.0.0.1:8000` 仍能正常使用，两者共享同一个 LLM 引擎和工具链，但对话历史各自独立。

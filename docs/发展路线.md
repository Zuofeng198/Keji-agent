# 科吉 AI 助手 — 未来发展路线

## 一、当前定位

科吉 = 企业级桌面 AI 助理，对标飞书/钉钉 AI，但：
- 更轻量：单机部署，无需复杂配置
- 本地优先：支持 Ollama 本地模型，数据不外传
- 工具链完整：文档/表格/PPT/OCR/邮件/知识库/文件管理

## 二、技术方向

### 阶段 1：基础夯实（当前 → v1.1）

**多引擎支持**
- [x] nanobot AgentRunner 引擎（已集成）
- [x] OpenAI 兼容 API（DeepSeek 等）
- [x] Ollama 本地模型
- [ ] Anthropic Claude 原生 SDK 接入
- [ ] Gemini API 接入
- [ ] 模型热切换（对话中切模型）

**工具系统重构**
- [x] nanobot ToolRegistry 工具注册
- [ ] MCP 协议支持（对接外部工具市场）
- [ ] 工具执行超时/重试/取消机制
- [ ] 工具调用成本统计

**记忆系统**
- [x] 会话持久化（JSONL 文件）
- [ ] 长期记忆（用户偏好/习惯）
- [ ] 跨会话上下文继承
- [ ] 记忆可视化/编辑界面

### 阶段 2：智能增强（v1.1 → v1.5）

**多模态**
- [ ] 图片理解（不局限于 OCR）
- [ ] 截图直接提问
- [ ] PDF 图表/流程图理解

**文件处理升级**
- [ ] 批量文件智能分类（AI 自动打标签）
- [ ] 跨格式转换（PDF→Word→Excel）
- [ ] 内容对比/差异检测
- [ ] 模板引擎（合同/报告模板自动填充）

**自动化工作流**
- [ ] 定时任务（每天自动生成报表）
- [ ] 文件夹监听（新文件自动处理）
- [ ] 多步工作流编排（UI 拖拽配置）
- [ ] 触发条件：文件到达/时间/消息

### 阶段 3：协作与集成（v1.5 → v2.0）

**多端**
- [ ] WebDAV/网盘集成
- [ ] 移动端（飞书/企微消息直接对话）
- [ ] 企业微信/钉钉深度集成
- [ ] 邮件客户端插件

**多人协作**
- [ ] 共享知识库
- [ ] 团队会话/任务分配
- [ ] 权限管理
- [ ] 操作审计日志

**企业集成**
- [ ] LDAP/OAuth 登录
- [ ] 企业 API 网关
- [ ] 数据看板（KPI 统计）
- [ ] SLA 监控告警

### 阶段 4：AI Native（v2.0+）

**自主 Agent**
- [ ] 长期目标驱动（"每周五自动汇总团队周报"）
- [ ] 工具自主学习（用户演示一次就学会）
- [ ] 多 Agent 协作（写作+审核+发送）
- [ ] Agent 市场（下载别人做好的 Agent）

**私有化部署**
- [ ] Docker 一键部署
- [ ] 离线完全可用
- [ ] 数据加密存储
- [ ] 合规审计

## 三、每周迭代节奏建议

```
周一：规划本周功能 + Bug fix
周二三四：开发
周五：测试 + 发布
周末：用户反馈收集
```

**建议工具链：**
- Issue 跟踪：GitHub Projects
- 文档：项目内 docs/ 目录
- 测试：pytest + 手动测试卡
- 发布：git tag + CHANGELOG
